# FPGA-face-recognition
# FPGA-Based Human Face Detection System

A Verilog HDL image processing system that detects human faces in images using RGB skin tone analysis and facial feature position detection. Designed for simulation with Icarus Verilog, with architecture suitable for synthesis onto real FPGA hardware.

---

## Overview

This system takes a bitmap image as input and determines whether a human face is present by:
1. Converting the image to a HEX pixel stream using MATLAB
2. Loading the pixel data into Verilog memory registers
3. Scanning every pixel for skin tone using RGB thresholds
4. Locating the bounding box around detected skin regions
5. Dividing the bounding box into thirds and searching for eyes, nose and mouth
6. Making a final decision based on all features being present and correctly positioned

The output is a processed BMP image with skin pixels highlighted in red and the detected face region outlined in green, along with a terminal report showing exactly which features were found or why detection failed.

## How To Use

### Step 1 — Prepare Your Image

Save your image as a `.bmp` file. Note the exact file path.

### Step 2 — Convert Image to HEX Using MATLAB

Open MATLAB and run the following script. Update the file paths to match your system:

```matlab
b = imread('/Users/YOUR_NAME/Desktop/image.bmp');

[HEIGHT, WIDTH, ~] = size(b);
disp([HEIGHT, WIDTH]);  % Note these values — you need them for Verilog

k = 1;
a = zeros(1, HEIGHT*WIDTH*3);

for i = HEIGHT:-1:1
    for j = 1:WIDTH
        a(k)   = b(i,j,1);  % R
        a(k+1) = b(i,j,2);  % G
        a(k+2) = b(i,j,3);  % B
        k = k + 3;
    end
end

fid = fopen('/Users/YOUR_NAME/Desktop/FPGA/kodim24.hex', 'wt');
fprintf(fid, '%x\n', a);
fclose(fid);
disp('HEX file created');
```

This loops through every pixel row by row (bottom to top, as BMP stores rows in reverse), extracts the R, G, B values separately, and writes each as a HEX byte on its own line.

### Step 3 — Update Verilog Parameters

Open `facerecognition` and update these lines at the top to match your image dimensions (printed by MATLAB in Step 2):

```verilog
parameter WIDTH  = 738;    // ← change to your image width
parameter HEIGHT = 1104;   // ← change to your image height
```

Also update the memory size on this line — it must equal `WIDTH × HEIGHT × 3`:

```verilog
reg [7:0] mem [0:2443775];  // ← change to (WIDTH * HEIGHT * 3) - 1
```

For example, for a 1200×1600 image: `1200 × 1600 × 3 - 1 = 5759999`

### Step 4 — Compile and Run

```bash
cd /Users/YOUR_NAME/Desktop/FPGA
iverilog -o sim.out facerecognition
vvp sim.out
```

## How the Detection Works

### Stage 1 — Skin Detection

Every pixel is checked against 11 RGB conditions:

| Condition | Purpose |
|---|---|
| `R > 95` | Eliminate very dark pixels / shadows |
| `G > 40` | Some green must be present |
| `B > 20` | Some blue must be present |
| `R > G` | Red must dominate — skin is warm toned |
| `R > B` | Red stronger than blue |
| `(R-G) > 15` | Enough red-green contrast |
| `(R-B) > 20` | Enough red-blue contrast |
| `(R+G+B) > 250` | Not in shadow |
| `R < 240` | Not overexposed / pure white |
| `G < 200` | Not washed out |
| `B < 170` | Not too blue |

All 11 conditions must pass for a pixel to be classified as skin. Skin pixels are counted and their positions tracked to build a bounding box.

The **skin ratio** (skin pixels ÷ total pixels) must exceed **3%** to proceed to feature detection.

### Stage 2 — Facial Feature Detection

The bounding box around all skin pixels is divided into three equal horizontal thirds:
┌─────────────────┐
│   TOP THIRD     │  ← Search for EYES here
├─────────────────┤
│  MIDDLE THIRD   │  ← Search for NOSE here
├─────────────────┤
│  BOTTOM THIRD   │  ← Search for MOUTH here
└─────────────────┘

**Eyes:** Dark pixels (brightness < 150) are counted in the left and right halves of the top third separately. Both halves must have 50–50,000 dark pixels, must be within 3× of each other in count (balanced), and the separation between the average eye positions must be 20%–80% of the face width.

**Nose:** Dark pixels (brightness < 200) are counted in the centre 33% of the face width within the middle third. Must have 50–100,000 dark pixels.

**Mouth:** Dark pixels counted in the centre 50% of the face width within the bottom third. Must have 50–100,000 dark pixels and the search width must be wider than the nose search width.

### Stage 3 — Final Decision
IF skin_ratio > 3%
AND both eyes found and balanced
AND nose found
AND mouth found
AND bounding box is a reasonable size
→ HUMAN FACE DETECTED
ELSE IF skin_ratio > 3%
→ SKIN DETECTED BUT NOT A HUMAN FACE
(reports exactly which feature failed)
ELSE
→ NO FACE DETECTED

The key differentiator from animals and landscapes is the **eye balance check** — a human face always has two roughly symmetrical eyes. A parrot, cat or landscape will have either no dark eye regions, or heavily unbalanced ones.

## Troubleshooting

| Problem | Solution |
|---|---|
| `Too many words in file` warning | Memory array size is too small — recalculate `WIDTH × HEIGHT × 3 - 1` and update line 11 |
| `No such file or directory` on compile | Wrong folder — run `cd` to navigate to the folder containing both `.v` and `.hex` files |
| `Skin ratio = 0.000000` | Wrong WIDTH/HEIGHT parameters, or HEX file not regenerated after image change |
| `Preprocessor failed` | Verilog file has a syntax error or wasn't saved correctly |
| Face not detected on clear photo | Skin thresholds may need loosening, or bounding box is dominated by clothing |

## Limitations

- Skin detection is sensitive to lighting — very dark or overexposed images may fail
- The bounding box is built from all skin pixels including clothing and background, not just the face
- Detection works best on forward-facing portraits
- BMP header in output is minimal — some image viewers may not open it correctly

---

## FPGA Synthesis Notes

The design is written as behavioural Verilog for simulation. For synthesis onto real FPGA hardware:

- Replace `$readmemh` with a BRAM (Block RAM) initialised from the image data
- Replace `$fwrite` output with GPIO pins, LED indicators, or HDMI/VGA display output
- The pixel-wise loop maps to a **state machine** or **pipelined datapath** in hardware
- Each skin detection condition becomes a hardware comparator — all 11 can run in parallel
- Add a camera interface (e.g. OV7670) for live real-time input
